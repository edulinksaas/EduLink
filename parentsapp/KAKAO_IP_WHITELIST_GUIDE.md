# 카카오 호출 허용 IP 주소 설정 가이드

## 호출 허용 IP 주소란?

카카오 API 키의 보안을 강화하기 위해, 특정 IP 주소에서만 API를 호출할 수 있도록 제한하는 기능입니다.

## 현재 프로젝트에서는 필요 없습니다!

**중요**: 현재 프로젝트는 Supabase를 통한 OAuth를 사용하므로, **호출 허용 IP 주소 설정은 필요하지 않습니다.**

### 이유:
1. **Supabase OAuth 사용**: Supabase 서버가 카카오 API를 호출합니다
2. **동적 IP**: Supabase 서버의 IP는 동적으로 변경될 수 있습니다
3. **리다이렉트 방식**: OAuth는 리다이렉트 방식으로 작동하므로 IP 제한이 필요 없습니다

## 언제 필요한가요?

호출 허용 IP 주소 설정이 필요한 경우:

1. **직접 API 호출**: 서버에서 직접 카카오 API를 호출하는 경우
2. **고정 IP 서버**: 고정된 IP 주소를 가진 서버에서 사용하는 경우
3. **보안 강화**: API 키 유출 시 추가 보안이 필요한 경우

## 설정 방법 (필요한 경우)

### 1단계: 설정 위치 찾기

1. Kakao Developers Console 접속
2. **내 애플리케이션** > 앱 선택
3. **앱 설정** > **앱 키** 메뉴로 이동
4. **호출 허용 IP 주소** 섹션 찾기

### 2단계: IP 주소 확인

#### Supabase 서버 IP (필요한 경우):

Supabase는 동적 IP를 사용하므로, 일반적으로 IP 화이트리스트를 설정할 수 없습니다.

#### 개발 환경 IP:

개발 중인 컴퓨터의 IP 주소를 확인:

**Windows:**
```cmd
ipconfig
```
IPv4 주소 확인 (예: 192.168.0.100)

**Mac/Linux:**
```bash
ifconfig
```
또는
```bash
ip addr
```

### 3단계: IP 주소 등록

호출 허용 IP 주소 섹션에서:

1. **IP 주소 추가** 또는 **+ 추가** 버튼 클릭
2. 다음 형식 중 하나로 입력:

#### 형식 1: 단일 IP
```
192.168.0.100
```

#### 형식 2: CIDR (서브넷)
```
192.168.0.0/24
```
(192.168.0.0 ~ 192.168.0.255 범위 허용)

#### 형식 3: Subnet Mask
```
192.168.0.0/255.255.255.0
```

### 4단계: 저장

1. IP 주소 입력 후 **등록** 또는 **저장** 클릭
2. 목록에 추가되었는지 확인

## 현재 프로젝트 설정 권장사항

### ✅ 권장: 설정하지 않음

**이유:**
- Supabase OAuth 사용 시 IP 제한 불필요
- Supabase 서버 IP는 동적이므로 설정 불가
- OAuth 리다이렉트 방식은 IP 제한과 무관

### ⚠️ 설정하려는 경우

개발 환경에서만 테스트하는 경우:

1. **개발용으로만 설정**:
   ```
   192.168.0.0/24  (로컬 네트워크 허용)
   ```

2. **주의사항**:
   - 프로덕션 환경에서는 설정하지 않는 것이 좋습니다
   - Supabase를 통한 OAuth는 IP 제한 없이 작동해야 합니다

## IP 주소 형식 예시

### 단일 IP
```
192.168.0.100
```

### CIDR 형식 (서브넷)
```
192.168.0.0/24      # 192.168.0.0 ~ 192.168.0.255
10.0.0.0/8          # 10.0.0.0 ~ 10.255.255.255
172.16.0.0/12       # 172.16.0.0 ~ 172.31.255.255
```

### Subnet Mask 형식
```
192.168.0.0/255.255.255.0
10.0.0.0/255.0.0.0
```

## 주의사항

### 1. 설정 시 주의할 점

- **모든 IP 허용**: 설정하지 않으면 모든 IP에서 호출 가능
- **IP 제한 설정**: 설정하면 지정된 IP에서만 호출 가능
- **잘못된 설정**: 잘못 설정하면 API 호출이 실패할 수 있음

### 2. Supabase 사용 시

- Supabase 서버 IP는 동적이므로 설정 불가
- OAuth 리다이렉트는 IP 제한과 무관
- **설정하지 않는 것을 권장**

### 3. 개발 환경

- 개발 중에는 설정하지 않는 것이 편리함
- 필요시 로컬 네트워크만 허용: `192.168.0.0/24`

## 체크리스트

호출 허용 IP 주소 설정:

- [ ] 현재 프로젝트에서는 **설정하지 않음** (권장)
- [ ] Supabase OAuth 사용 시 IP 제한 불필요
- [ ] 개발 환경에서만 테스트하는 경우 로컬 네트워크 허용 가능
- [ ] 프로덕션 환경에서는 설정하지 않는 것이 좋음

## 요약

### 현재 프로젝트:
- ✅ **설정하지 않음** (권장)
- Supabase OAuth 사용 시 IP 제한 불필요
- OAuth 리다이렉트 방식은 IP와 무관

### 설정이 필요한 경우:
- 직접 API 호출하는 서버가 있는 경우
- 고정 IP를 가진 서버에서 사용하는 경우
- 보안 강화가 필요한 경우

**핵심**: 현재 프로젝트(Supabase OAuth 사용)에서는 호출 허용 IP 주소 설정이 필요하지 않습니다!
