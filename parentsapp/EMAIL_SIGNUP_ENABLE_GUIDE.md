# 이메일 회원가입 활성화 가이드

## 오류 메시지
```
Email signups are disabled
```

이 오류는 Supabase에서 이메일 회원가입 기능이 비활성화되어 있을 때 발생합니다.

## 해결 방법

### 1단계: Supabase 대시보드 접속

1. [Supabase](https://supabase.com)에 접속
2. 프로젝트 선택

### 2단계: Authentication 설정으로 이동

1. 좌측 메뉴에서 **Authentication** 클릭
2. **Providers** 메뉴 클릭

### 3단계: Email 프로바이더 활성화

1. **Email** 프로바이더 찾기
2. **Enable Email provider** 토글을 **ON**으로 변경
3. **Enable email confirmations** 설정 확인:
   - **ON**: 이메일 인증이 필요함 (회원가입 후 이메일 확인 필요)
   - **OFF**: 이메일 인증 없이 바로 로그인 가능 (개발/테스트용 권장)
4. **Save** 버튼 클릭

### 4단계: 추가 설정 (선택 사항)

#### 이메일 인증 없이 사용하기 (개발/테스트용)

1. **Authentication** > **Settings** 메뉴로 이동
2. **Enable email confirmations** 옵션을 **OFF**로 설정
3. **Save** 버튼 클릭

> **참고**: 프로덕션 환경에서는 보안을 위해 이메일 인증을 활성화하는 것을 권장합니다.

#### 이메일 없이도 회원가입 허용 (선택 사항)

1. **Authentication** > **Settings** 메뉴로 이동
2. **Allow users to sign up without an email** 옵션 확인
3. 필요에 따라 활성화/비활성화

### 5단계: 앱 재시작

설정 변경 후 앱을 재시작하세요:

```bash
cd app
npm start
```

그리고 앱을 다시 로드하세요 (개발 서버에서 `r` 키 누르기).

## 확인 사항

### Supabase 설정:
- [ ] Email provider: **Enabled** ✅
- [ ] Enable email confirmations: **OFF** (개발용) 또는 **ON** (프로덕션용)
- [ ] Save 버튼 클릭 완료

### 테스트:
- [ ] 앱 재시작 완료
- [ ] 이메일 회원가입 시도
- [ ] 회원가입 성공 확인

## 이메일 인증이 활성화된 경우

만약 **Enable email confirmations**가 **ON**으로 설정되어 있다면:

1. 회원가입 후 이메일로 확인 링크가 전송됩니다
2. 이메일의 확인 링크를 클릭해야 로그인이 가능합니다
3. 확인 전까지는 로그인할 수 없습니다

### 개발 중에는 이메일 인증 비활성화 권장

개발 및 테스트 단계에서는 이메일 인증을 비활성화하는 것이 편리합니다:
- 즉시 로그인 가능
- 이메일 확인 과정 없이 테스트 가능

## 문제 해결

### 여전히 오류가 발생하는 경우

1. **Supabase 대시보드에서 설정 확인**
   - Email provider가 정말 **Enabled**인지 확인
   - 페이지를 새로고침하고 다시 확인

2. **앱 완전 재시작**
   - 개발 서버 완전 종료
   - 앱 완전 종료
   - 다시 시작

3. **환경 변수 확인**
   - `.env` 파일에 올바른 Supabase URL과 키가 있는지 확인
   - 앱 재시작 후에도 적용되는지 확인

4. **Supabase 프로젝트 확인**
   - 올바른 프로젝트를 선택했는지 확인
   - 다른 프로젝트의 설정을 확인하고 있지 않은지 확인

## 참고

- 이메일 회원가입은 Supabase의 기본 인증 방법입니다
- 다른 소셜 로그인(구글, 카카오 등)과 함께 사용할 수 있습니다
- 이메일 인증을 활성화하면 보안이 강화되지만, 사용자 경험이 약간 복잡해질 수 있습니다
