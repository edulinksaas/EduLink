# 카카오계정(이메일) 권한 설정 방법

## 현재 상태
- **카카오계정(이메일)**: 권한 없음 ❌
- 이것이 KOE205 에러의 원인입니다!

## 해결 방법

### 방법 1: 동의 항목 활성화 (권장)

#### 1단계: 설정 버튼 클릭
1. **카카오계정(이메일)** 행의 **설정** 버튼 클릭
   - 현재는 설정 버튼이 보이지 않을 수 있습니다
   - 다른 행(닉네임, 프로필 사진)의 설정 버튼을 참고하세요

#### 2단계: 동의 항목 활성화
1. 설정 창이 열리면:
   - **필수 동의** 또는 **선택 동의** 선택
   - **선택 동의**를 권장합니다 (사용자가 선택할 수 있음)
2. **저장** 또는 **적용** 클릭

#### 3단계: 상태 확인
- 상태가 "권한 없음" → "선택 동의" 또는 "이용 중 동의"로 변경되는지 확인

### 방법 2: 동의 항목 추가하기

설정 버튼이 없는 경우:

1. **동의 항목 추가** 또는 **+ 추가** 버튼 찾기
2. **카카오계정(이메일)** 또는 **account_email** 선택
3. **선택 동의**로 설정
4. **저장** 클릭

### 방법 3: Supabase 설정 변경 (이메일 불필요한 경우)

이메일이 필요하지 않다면:

1. Supabase 대시보드 > **Authentication** > **Providers** > **Kakao**
2. **Allow users without an email** 토글을 **ON**으로 설정
3. **Save** 클릭

이렇게 하면 이메일 없이도 로그인할 수 있습니다.

## 예상되는 결과

설정 완료 후:
- ✅ 상태: "권한 없음" → "선택 동의" 또는 "이용 중 동의"
- ✅ 설정 버튼이 나타남
- ✅ KOE205 에러 해결

## 단계별 가이드

### Kakao Developers Console에서:

1. **동의항목 페이지 확인**
   - 현재 페이지가 동의항목 설정 페이지인지 확인

2. **카카오계정(이메일) 행 찾기**
   - 테이블에서 "카카오계정(이메일)" 행 찾기
   - ID: `account_email`
   - 상태: "권한 없음" (현재)

3. **설정 방법 선택**
   - 방법 A: 설정 버튼이 있다면 클릭
   - 방법 B: 동의 항목 추가 버튼 사용
   - 방법 C: 페이지 상단의 "동의 항목 추가" 메뉴 사용

4. **동의 항목 활성화**
   - **선택 동의**로 설정 (권장)
   - 또는 **필수 동의**로 설정

5. **저장**
   - 변경사항 저장
   - 상태가 변경되었는지 확인

## 요약

**문제**: 카카오계정(이메일)이 "권한 없음" 상태

**해결**:
1. ✅ 동의 항목 설정에서 **카카오계정(이메일)** 활성화
2. ✅ **선택 동의** 또는 **필수 동의**로 설정
3. ✅ 저장 후 테스트

**또는**:
- Supabase에서 **Allow users without an email** 활성화
